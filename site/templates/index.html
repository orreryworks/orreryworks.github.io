Software architecture diagrams
from simple text.

Orrery is a diagram language for creating component and sequence diagrams. Write text, get SVGs.

cargo install orrery-cli Get Started GitHub

See it in action

input.orr
diagram component;

type Service = Rectangle [fill_color="#e6f3ff"];
type Database = Rectangle [fill_color="#e0f0e0"];

api as "API Gateway": Service;
auth as "Auth Service": Service;
db as "Users DB": Database;

api -> auth: "Verify";
auth -> db: "Query";
db -> auth: "Result";
auth -> api: "Token";
→
output.svg

Why Orrery?

Text-based & version-controllable

Diagrams live in .orr files alongside your code. Diff, review, and merge them like any source file.

Component & sequence diagrams

Model system architecture with component diagrams and interaction flows with sequence diagrams.

Type system for reusable styles

Define types once, apply everywhere. Custom shapes, colors, and styling inherit through a composable type system.

Clean SVG output

Renders to standard SVG. Embed in documentation, wikis, or anywhere that accepts images.
